# 贡献指南

感谢你对 **Simulink 无人机动力学仿真** 教学项目的关注！我们欢迎各种形式的贡献，包括但不限于内容修正、新增章节、示例模型优化和文档改进。

## 如何贡献

### 提交 Issue

如果你发现了以下问题，请通过 [Issue](../../issues) 反馈：

- **内容错误** — 公式推导有误、参数描述不准确、仿真步骤遗漏等
- **拼写/格式问题** — 错别字、Markdown 渲染异常、图片缺失等
- **功能建议** — 希望新增某个知识点或补充特定控制算法的讲解
- **工具箱兼容性** — 某个 Simulink 模型在特定 MATLAB 版本下无法运行

**Issue 模板：**

```markdown
## 问题类型
- [ ] 内容错误
- [ ] 拼写/格式问题
- [ ] 功能建议
- [ ] 兼容性问题

## 所在章节
<!-- 例如：第 3 章 第 2 节 -->

## 问题描述
<!-- 请尽可能详细地描述问题 -->

## 环境信息（如适用）
- MATLAB 版本：
- Simulink 版本：
- 操作系统：
```

### 提交 Pull Request (PR)

1. **Fork 本仓库** 到你的 GitHub 账户
2. **创建特性分支**：`git checkout -b feature/你的特性名称`
3. **进行修改** 并确保符合下方的内容规范
4. **提交更改**：`git commit -m "简要描述你的修改"`
5. **推送分支**：`git push origin feature/你的特性名称`
6. **发起 Pull Request**，在 PR 描述中说明修改内容和原因

**PR 建议：**

- 保持每个 PR 聚焦于单一主题，避免将多个不相关的修改混在一起
- 如果修改较大，建议先开 Issue 讨论方案
- PR 标题应简洁明了，例如：`fix: 修正第5章PID参数公式错误` 或 `feat: 新增第11章编队控制`

## 内容规范

### 语言要求

- **正文语言**：以中文（简体）为主要撰写语言
- **专业术语**：首次出现时保留英文原文并附中文翻译，格式为 `中文翻译（English Term）`，后续可直接使用中文
- **代码注释**：Simulink 模型注释和 MATLAB 脚本注释可使用中英文混合
- **示例**：
  - 正确："四元数（Quaternion）是一种用于表示三维旋转的数学工具。"
  - 正确："欧拉角（Euler Angles）存在万向节锁（Gimbal Lock）问题。"
  - 避免：整段使用英文撰写（除非引用原始论文/官方文档）

### 文档格式

- **Markdown 规范**：
  - 使用 ATX 风格标题（`#`、`##`、`###`）
  - 代码块使用三反引号并标注语言（````matlab`、````python`、````text`）
  - 数学公式使用 LaTeX 语法，行内公式用 `$...$`，独立公式用 `$$...$$`
  - 表格使用 Markdown 原生语法

- **文件命名**：
  - 章节文件：`docs/chapter-XX-标题.md`，例如 `docs/chapter-03-quadrotor-dynamics.md`
  - 图片文件：`docs/images/chapter-XX/描述性名称.png`
  - Simulink 模型：`models/功能描述.slx`，例如 `models/quadrotor_pid.slx`
  - MATLAB 脚本：`scripts/功能描述.m`

- **章节结构**：每章应包含以下标准段落：

  ```markdown
  # 第X章 标题

  ## 本章目标
  <!-- 3-5 个学习目标，使用动词开头 -->

  ## 理论基础
  <!-- 数学推导、物理原理 -->

  ## Simulink 建模
  <!-- 分步骤的模型搭建指南 -->

  ## 仿真与分析
  <!-- 运行仿真、解读结果 -->

  ## 进阶拓展
  <!-- 可选：更深入的内容或变体 -->

  ## 本章小结
  <!-- 关键知识点回顾 -->

  ## 练习题
  <!-- 2-3 道动手练习 -->

  ## 参考文献
  <!-- 引用的论文、书籍、官方文档 -->
  ```

### Simulink 模型规范

- 模型应使用 Simulink **R2023b** 及以上版本保存（`.slx` 格式）
- 模型中应包含清晰的 **注释**（Annotation），说明各子系统的功能
- 使用 **Subsystem** 对功能模块进行封装，保持模型层次清晰
- 在模型属性中填写 `Description` 字段，简要说明模型用途
- 提供配套的 MATLAB 初始化脚本（`setup_xxx.m`），用于加载参数

### 数学公式规范

- 使用标准 LaTeX 数学符号
- 向量用粗体或箭头表示，例如 $\mathbf{v}$ 或 $\vec{v}$
- 矩阵用大写粗体表示，例如 $\mathbf{R}$
- 坐标变换矩阵注明源坐标系和目标坐标系，例如 ${}^{B}\mathbf{R}_{N}$ 表示从导航系到机体系的旋转矩阵

## 质量标准

所有贡献内容需满足以下标准：

| 维度 | 要求 |
|------|------|
| **准确性** | 数学推导正确，物理概念准确，仿真结果可复现 |
| **完整性** | 包含完整的推导过程，不跳步；Simulink 模型可独立运行 |
| **可读性** | 逻辑清晰，循序渐进，适合目标读者的知识水平 |
| **一致性** | 术语、符号、格式与已有文档保持统一 |
| **可复现性** | 所有仿真结果应能在指定 MATLAB 版本下复现，注明所需工具箱 |

## 审核流程

1. 提交 PR 后，维护者会在 **3 个工作日内** 进行初审
2. 初审通过后，进入内容审核阶段，重点检查技术准确性
3. 如有修改意见，会在 PR 中以 Review Comment 形式提出
4. 修改通过后合并到主分支

## 行为准则

- 尊重每一位贡献者，保持友善和专业的沟通
- 对建设性批评持开放态度
- 不发布与项目无关的内容
- 遵守 [MIT 许可证](LICENSE) 条款

## 联系方式

- 提交 [Issue](../../issues) 进行公开讨论
