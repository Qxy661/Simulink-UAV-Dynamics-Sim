# 子系统封装与模块化

> 预计阅读：20 分钟 | 前置知识：Simulink 基础操作、已搭建过基础 Simulink 模型

当模型规模增大时，良好的模块化设计是保持模型可维护性、可复用性的关键。本文介绍 Simulink 中子系统封装、Mask 编辑器、Model Reference 等核心工具。

---

## 1. 子系统 (Subsystem) 创建

### 1.1 为什么需要子系统

| 没有子系统 | 有子系统 |
|-----------|---------|
| 所有模块平铺在画布上 | 层次化组织，逻辑清晰 |
| 信号线交叉密集 | 每层只显示关键连接 |
| 难以定位问题 | 可逐层排查 |
| 无法复用 | 可封装后复用 |

### 1.2 创建方法

**方法一：选中模块后封装**
```
1. 选中需要组合的模块 (Ctrl+点击 或 框选)
2. 右键 > Create Subsystem from Selection
   或按 Ctrl+G
3. 自动创建子系统，保留外部连接
```

**方法二：从空子系统开始**
```
1. 从 Library Browser 拖入 Subsystem 模块
2. 双击进入子系统
3. 在子系统内部搭建模型
```

### 1.3 子系统层次结构示例

```
quadrotor_6dof (顶层)
├── Motor_Model (子系统)
│   ├── Motor_1 (子系统)
│   │   ├── PWM_to_RPM (Gain)
│   │   ├── Motor_Dynamics (Transfer Fcn)
│   │   └── Saturation
│   ├── Motor_2 (子系统)
│   ├── Motor_3 (子系统)
│   └── Motor_4 (子系统)
├── Force_Torque (子系统)
│   ├── Thrust_Calculation (子系统)
│   ├── Moment_Calculation (子系统)
│   └── Gravity_Transform (子系统)
├── Dynamics_6DOF (子系统)
│   ├── Translation (子系统)
│   └── Rotation (子系统)
└── Visualization (子系统)
    ├── Position_Scope
    ├── Attitude_Scope
    └── Motor_Scope
```

## 2. Mask Editor (封装编辑器)

Mask 让子系统拥有自定义参数对话框，用户无需进入子系统内部即可修改参数。

### 2.1 创建 Mask

```
1. 右键子系统 > Mask > Create Mask (或 Ctrl+M)
2. 打开 Mask Editor 对话框
```

### 2.2 Mask Editor 四个标签页

| 标签页 | 功能 | 常用设置 |
|--------|------|----------|
| **Icon & Ports** | 自定义图标和端口标签 | 绘制图标、显示参数值 |
| **Parameters & Dialog** | 定义参数界面 | 添加参数控件 |
| **Initialization** | 初始化代码 | 参数验证、计算派生量 |
| **Documentation** | 帮助文档 | 模块描述和帮助文本 |

### 2.3 参数控件类型

| 控件类型 | 图标 | 用途 | 示例 |
|----------|------|------|------|
| **Edit** | 文本框 | 输入数值或表达式 | 质量 m = 1.5 |
| **Popup** | 下拉菜单 | 从预定义选项中选择 | 求解器类型 |
| **Checkbox** | 复选框 | 开关选项 | 是否启用陀螺效应 |
| **Slider** | 滑块 | 范围内取值 | PID 增益调参 |
| **Dial** | 旋钮 | 角度类参数 | 偏航角设定 |
| **Promote** | 传递参数 | 将内部参数暴露出来 | Integrator 初始条件 |

### 2.4 实例：Motor-Propeller 子系统 Mask

创建一个电机-螺旋桨子系统的 Mask：

**Parameters & Dialog 设置：**

| Prompt | Name | Type | 默认值 | 说明 |
|--------|------|------|--------|------|
| 推力系数 Ct | `Ct` | Edit | `1.48e-07` | 单位 N/(rpm)^2 |
| 力矩系数 Cm | `Cm` | Edit | `2.31e-09` | 单位 N·m/(rpm)^2 |
| 时间常数 tau | `tau` | Edit | `0.03` | 单位 s |
| 最大转速 | `max_rpm` | Edit | `10000` | 单位 rpm |
| 旋转方向 | `direction` | Popup | `CW` | CW / CCW |

**Initialization 代码：**

```matlab
% 参数验证
if Ct <= 0
    error('推力系数 Ct 必须为正数');
end
if tau <= 0
    error('时间常数 tau 必须为正数');
end
% 派生参数计算
max_thrust = Ct * max_rpm^2;
hover_rpm = sqrt(m * g / 4 / Ct);
```

### 2.5 Icon & Ports 自定义

```
在 Icon drawing commands 中输入:

% 显示参数值
disp(sprintf('Ct=%.2e', Ct));
disp(sprintf('tau=%.3f s', tau));

% 或绘制图形
port_label('input', 1, 'PWM');
port_label('output', 1, 'rpm');
port_label('output', 2, 'thrust');
color('blue');
text(0.5, 0.5, 'Motor', 'HorizontalAlignment', 'center');
```

## 3. Model Reference vs Subsystem

| 特性 | Subsystem | Model Reference |
|------|-----------|-----------------|
| 存储方式 | 嵌入在父模型中 | 独立 .slx 文件 |
| 加载方式 | 随父模型一起加载 | 按需加载(加速器模式) |
| 编译方式 | 每次都重新编译 | 可缓存编译结果 |
| 团队协作 | 多人无法同时编辑 | 独立文件，Git 友好 |
| 版本控制 | 整个文件变更 | 只有引用文件变更 |
| 代码生成 | 整体生成 | 独立生成，可复用 |
| 适用场景 | 小型、简单子系统 | 大型、复用频率高的模块 |
| 初始化 | 直接继承父模型参数 | 需要传递参数 |
| 仿真速度 | 深层嵌套时慢 | 加速器模式下快 |

### 3.1 创建 Model Reference

```matlab
% 创建独立的电机模型文件
new_system('motor_prop_model')
open_system('motor_prop_model')
% ... 搭建模型 ...
save_system('motor_prop_model')

% 在主模型中引用
% Library Browser > Model > Model Reference
% 设置 Model name: motor_prop_model
```

### 3.2 Model Reference 配置

| 参数 | 推荐设置 | 说明 |
|------|----------|------|
| Simulation mode | Accelerator | 首次编译后缓存 |
| Code interface | Reusable function | 代码可复用 |
| Pass fixed-size scalar root inputs by value | ✅ | 提升性能 |

## 4. Variant Subsystems (变体子系统)

变体子系统允许在同一位置放置多个备选实现，通过切换条件选择激活哪一个。

### 4.1 使用场景

```
场景: 同一架 UAV 可以使用不同的动力学模型进行仿真

┌──────────────────────┐
│   Variant Subsystem  │
│                      │
│  ├── Variant 1: 简化模型 (质点模型)
│  ├── Variant 2: 6DOF 模型
│  └── Variant 3: Simscape 多体模型
│                      │
│  激活条件: VariantControl
└──────────────────────┘
```

### 4.2 创建方法

```
1. 从 Library Browser 拖入 Variant Subsystem
2. 右键 > Block Parameters
3. 添加 Variant choices (变体选项)
4. 为每个变体指定子系统
5. 设置 Variant control 变量
```

### 4.3 配置示例

```matlab
% 在 MATLAB 工作区定义变体控制变量
dynamics_variant = 2;  % 1=简化, 2=6DOF, 3=Simscape
```

在 Variant Subsystem 中：

| Variant | Condition | 子系统 |
|---------|-----------|--------|
| Simple | `dynamics_variant == 1` | point_mass_model |
| 6DOF | `dynamics_variant == 2` | six_dof_model |
| Simscape | `dynamics_variant == 3` | simscape_model |

## 5. Bus Objects (总线对象)

Bus 对象用于定义结构化的信号类型，类似于 C 语言中的 struct。

### 5.1 定义 Bus 对象

```matlab
% 在 MATLAB 脚本或 Simulink Data Dictionary 中定义
% 定义 UAV 状态总线
elems(1) = Simulink.BusElement;
elems(1).Name = 'position';
elems(1).Dimensions = 3;
elems(1).DataType = 'double';

elems(2) = Simulink.BusElement;
elems(2).Name = 'velocity';
elems(2).Dimensions = 3;
elems(2).DataType = 'double';

elems(3) = Simulink.BusElement;
elems(3).Name = 'attitude';
elems(3).Dimensions = 3;   % [phi, theta, psi]
elems(3).DataType = 'double';

elems(4) = Simulink.BusElement;
elems(4).Name = 'angular_rate';
elems(4).Dimensions = 3;   % [p, q, r]
elems(4).DataType = 'double';

UAV_StateBus = Simulink.Bus;
UAV_StateBus.Elements = elems;
```

### 5.2 使用 Bus 对象

```
Bus Creator 模块:
  - Output data type: Bus: UAV_StateBus
  - 输入端口自动匹配 Bus 定义的元素

Bus Selector 模块:
  - 输出端口可选择 Bus 中的任意元素
  - 如只选择 position 和 velocity
```

### 5.3 Bus 对象的优势

| 优势 | 说明 |
|------|------|
| 类型安全 | 编译时检查信号类型和维度 |
| 接口清晰 | 子系统输入输出用 Bus 定义，一目了然 |
| 减少连线 | 一根 Bus 线传递多个信号 |
| 自动文档 | Bus 定义即信号文档 |

## 6. Data Dictionary (数据字典)

数据字典 (.sldd 文件) 用于集中管理模型参数、Bus 对象、枚举类型等数据定义。

### 6.1 创建数据字典

```matlab
% 创建新的数据字典
dd = Simulink.data.dictionary.create('UAV_DataDict.sldd');

% 打开数据字典编辑器
Simulink.data.dictionary.open('UAV_DataDict.sldd');
```

### 6.2 数据字典内容组织

```
UAV_DataDict.sldd
├── Design Data (设计数据)
│   ├── params           (结构体: 物理参数)
│   ├── UAV_StateBus     (Bus 对象)
│   ├── Motor_Bus        (Bus 对象)
│   └── ControlMode      (枚举类型)
├── Configurations (配置)
│   ├── solver_config    (求解器配置)
│   └── code_gen_config  (代码生成配置)
└── Other Data
    └── signal_names     (信号名列表)
```

### 6.3 关联模型与数据字典

```
1. 打开 Model Explorer (Ctrl+Shift+I)
2. 选择模型根节点
3. 在 External Data 中选择数据字典文件
4. 应用后，模型中的参数引用将指向数据字典
```

### 6.4 数据字典 vs 工作区变量

| 特性 | 工作区变量 | Data Dictionary |
|------|-----------|-----------------|
| 持久性 | 关闭 MATLAB 后丢失 | .sldd 文件持久存储 |
| 作用域 | 全局，可能冲突 | 与模型绑定 |
| 版本控制 | 无法追踪 | 文件可 Git 管理 |
| 多人协作 | 冲突风险高 | 可锁定、合并 |
| 参数查找 | 需要加载脚本 | 自动关联 |

## 7. 命名规范与文档

### 7.1 模型命名规范

| 元素 | 命名规则 | 示例 |
|------|----------|------|
| 模型文件 | 小写 + 下划线 | `quadrotor_6dof.slx` |
| 子系统 | 大驼峰 | `MotorModel`, `ForceTorque` |
| 信号线 | 小写 + 下划线 | `position_cmd`, `motor_rpm` |
| 参数 | 小写 + 下划线 | `arm_length`, `max_rpm` |
| Bus 对象 | 大驼峰 + Bus 后缀 | `UAV_StateBus` |
| 常量 | 全大写 + 下划线 | `MAX_RPM`, `GRAVITY` |

### 7.2 子系统文档

每个关键子系统应添加注释：

```
双击子系统空白处 > 添加 Annotation:
┌─────────────────────────────────────────────┐
│  Motor Model Subsystem                      │
│  功能: 将 PWM 指令转换为电机转速              │
│  输入: PWM (4x1, 0~1000)                     │
│  输出: rpm (4x1), thrust (4x1)               │
│  模型: 一阶惯性环节 1/(τs+1)                  │
│  作者: xxx    日期: 2026-05-11               │
└─────────────────────────────────────────────┘
```

### 7.3 信号线命名

```
良好实践:
  ✅ 双击信号线添加名称 "motor_rpm"
  ✅ 关键信号启用 Signal Logging
  ✅ 使用 Bus 传递相关信号组

避免:
  ✗ 不命名信号线，后期无法追踪
  ✗ 使用无意义的名称如 "sig1", "out2"
  ✗ 过长的名称 (>30 字符)
```

## 8. 模型层次化设计最佳实践

### 8.1 设计原则

| 原则 | 说明 |
|------|------|
| **单一职责** | 每个子系统只做一件事 |
| **接口清晰** | 输入输出用 Bus 定义，明确类型和维度 |
| **层次合理** | 不超过 4~5 层嵌套 |
| **参数集中** | 使用数据字典或参数脚本管理参数 |
| **可替换性** | 使用 Variant Subsystem 支持多种实现 |
| **可测试性** | 每个子系统可独立仿真验证 |

### 8.2 推荐的 UAV 模型层次

```
Level 0: UAV_Top (顶层)
├── Level 1: Plant (被控对象)
│   ├── Level 2: Motor_Model
│   ├── Level 2: Aero_Model
│   ├── Level 2: Gravity_Model
│   └── Level 2: Dynamics_6DOF
│       ├── Level 3: Translation
│       └── Level 3: Rotation
├── Level 1: Controller (控制器)
│   ├── Level 2: Attitude_Controller
│   │   ├── Level 3: Roll_PID
│   │   ├── Level 3: Pitch_PID
│   │   └── Level 3: Yaw_PID
│   ├── Level 2: Position_Controller
│   └── Level 2: Motor_Mixer
├── Level 1: Environment (环境)
│   ├── Level 2: Wind_Model
│   ├── Level 2: Gravity_Model
│   └── Level 2: Atmosphere_Model
└── Level 1: Visualization (可视化)
    ├── Level 2: Scope_Group
    └── Level 2: Logging
```

### 8.3 模块化拆分检查清单

- [ ] 每个子系统是否有明确的输入输出定义？
- [ ] 子系统之间是否通过 Bus 传递信号？
- [ ] 是否有独立的参数管理脚本或数据字典？
- [ ] 每个子系统是否可独立测试？
- [ ] 命名是否遵循统一规范？
- [ ] 关键子系统是否有 Mask 参数对话框？
- [ ] 是否使用了 Model Reference 用于高频复用的模块？
- [ ] 层次深度是否控制在 4~5 层以内？

## 9. 实例：创建可复用的电机-螺旋桨子系统

### 9.1 目标

创建一个通用的 Motor-Propeller 子系统，支持：
- 参数化配置（推力系数、力矩系数等）
- 顺时针/逆时针可选
- 输出推力和反扭矩
- 可独立测试

### 9.2 子系统结构

```
┌─────────────────────────────────────────────────────┐
│  Motor_Prop (Subsystem with Mask)                    │
│                                                     │
│  输入:                                              │
│    port 1: pwm_cmd (0~1000)                         │
│                                                     │
│  输出:                                              │
│    port 1: thrust (N)                               │
│    port 2: reaction_torque (N·m)                    │
│    port 3: rpm (rpm)                                │
│                                                     │
│  内部:                                              │
│  pwm_cmd ──>[Gain rpm_per_pwm]──> rpm_cmd           │
│  rpm_cmd ──>[Transfer Fcn 1/(τs+1)]──> rpm         │
│  rpm ──>[Saturation 0~max_rpm]──> rpm_limited       │
│  rpm_limited ──>[Product u.*u]──> rpm_squared       │
│  rpm_squared ──>[Gain Ct]──> thrust                 │
│  rpm_squared ──>[Gain Cm*dir]──> reaction_torque    │
│                                                     │
│  Mask 参数: Ct, Cm, tau, max_rpm, direction         │
└─────────────────────────────────────────────────────┘
```

### 9.3 独立测试

```matlab
% 测试脚本 test_motor_prop.m
% 设置参数
Ct = 1.48e-07;
Cm = 2.31e-09;
tau = 0.03;
max_rpm = 10000;

% 阶跃输入测试
pwm_step = 500;  % 50% 油门
sim('motor_prop_test');

% 验证稳态值
expected_rpm = pwm_step * max_rpm / 1000;
expected_thrust = Ct * expected_rpm^2;
fprintf('期望转速: %.0f rpm\n', expected_rpm);
fprintf('期望推力: %.2f N\n', expected_thrust);
```

## 思考题

1. **Subsystem 和 Model Reference 在什么场景下应该选择哪个？请从仿真性能、团队协作、代码生成三个角度分析。**

2. **如果一个 UAV 项目需要同时支持四旋翼和六旋翼两种构型，如何使用 Variant Subsystem 实现动力学模型的切换？请给出设计思路。**

3. **Bus 对象和 Mux 都可以将多个信号合并传递，它们有什么本质区别？在 UAV 模型中推荐使用哪种？为什么？**

4. **Data Dictionary 相比在 MATLAB 脚本中定义参数有什么优势？在团队协作场景下，如何避免多人同时修改 Data Dictionary 造成冲突？**

5. **在创建 Mask 时，Initialization 代码中可以做什么事情？请举出至少三个实际应用场景。**

<details>
<summary>参考答案</summary>

**1.** 从三个角度分析：
- **仿真性能**：Subsystem 每次都重新编译，深层嵌套时较慢；Model Reference 使用 Accelerator 模式可缓存编译结果，大型模型中优势明显。
- **团队协作**：Subsystem 嵌入父模型，多人无法同时编辑同一文件；Model Reference 是独立 .slx 文件，不同人可同时修改不同模块。
- **代码生成**：Subsystem 生成的代码与父模型耦合；Model Reference 可独立生成代码，便于模块复用和单元测试。
推荐：简单、不常复用的模块用 Subsystem；核心、可复用的模块（如电机模型、控制器）用 Model Reference。

**2.** 设计思路：
- 创建一个 Variant Subsystem `DynamicsModel`
- Variant 1: `QuadrotorDynamics` (4 个电机输入)
- Variant 2: `HexrotorDynamics` (6 个电机输入)
- 顶层输入通过变体条件路由到对应的子系统
- 使用 `rotor_count` 变量控制：`rotor_count == 4` 激活四旋翼，`rotor_count == 6` 激活六旋翼
- 控制器部分也需要对应的 Variant Subsystem
- Motor Mixer 子系统根据构型选择不同的混控矩阵

**3.** 本质区别：
- Mux 只是将信号打包成向量，不保留信号语义（名称、数据类型、维度信息），访问时需要按索引
- Bus 对象保留完整的信号定义（名称、类型、维度），是强类型的结构化信号
在 UAV 模型中推荐使用 Bus：(1) 信号语义清晰，便于维护；(2) 编译时类型检查，减少错误；(3) Bus Selector 按名称选择，不受顺序影响；(4) 代码生成时映射为结构体，更易移植。

**4.** Data Dictionary 优势：(1) 持久化存储，不依赖工作区；(2) 与模型绑定，避免全局变量冲突；(3) 版本控制友好，.sldd 文件可 Git 管理；(4) 支持锁定机制。
团队协作避免冲突的方法：(1) 将 Data Dictionary 拆分为多个文件（如 `params.sldd`、`bus_defs.sldd`），不同人负责不同文件；(2) 使用 Git 的分支管理，修改后合并；(3) 设置只读保护，关键参数由专人维护；(4) 使用 Simulink 的数据字典锁定功能。

**5.** Initialization 代码的三个应用场景：
- (1) **参数验证**：检查参数范围合理性（如 `Ct > 0`、`tau > 0`），不合理时报错提示
- (2) **派生参数计算**：根据 Mask 参数计算中间变量（如根据 Ct、m、g 计算悬停转速）
- (3) **条件初始化**：根据一个 Popup 参数（如电机类型）自动设置相关的 Ct、Cm 等默认值
其他场景：单位转换、参数一致性检查、打印配置信息到诊断窗口。

</details>
