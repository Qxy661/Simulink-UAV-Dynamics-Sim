## 2. 编队控制架构

### 2.1 领航-跟随 (Leader-Follower)

最直观的编队策略：一架无人机作为领航者 (Leader)，其余作为跟随者 (Follower)。

```
         ┌──────────┐
         │  Leader  │ ──── 全局轨迹跟踪
         │  UAV #0  │
         └────┬─────┘
              │ 相对位置测量/通信
      ┌───────┼───────┐
      ▼       ▼       ▼
  ┌──────┐┌──────┐┌──────┐
  │Follower││Follower││Follower│
  │UAV #1 ││UAV #2 ││UAV #3 │
  └──────┘└──────┘└──────┘
```

**控制律：**

$$
\mathbf{u}_i = \mathbf{u}_{leader}(t) + k_p(\mathbf{d}_i^{des} - (\mathbf{p}_i - \mathbf{p}_{leader})) + k_d(\mathbf{0} - (\mathbf{v}_i - \mathbf{v}_{leader}))
$$

其中 $\mathbf{d}_i^{des}$ 是第 $i$ 架跟随者相对于领航者的期望偏移。

**优缺点：**

| 优点 | 缺点 |
|------|------|
| 实现简单直观 | 领航者是单点故障 |
| 跟随者只需知道领航者信息 | 领航者机动时跟随者反应延迟 |
| 便于人工指定编队形状 | 无法扩展到大规模编队 |

### 2.2 虚拟结构 (Virtual Structure)

将编队视为一个刚体，由虚拟中心控制整体运动。

```
  ┌─────────────────────────────────┐
  │        虚拟刚体 (Virtual)        │
  │                                 │
  │   ○ (UAV #1)     ○ (UAV #2)   │
  │           ○ (中心)              │
  │   ○ (UAV #3)     ○ (UAV #4)   │
  │                                 │
  └─────────────────────────────────┘
  整体按虚拟中心轨迹运动
```

**控制流程：**
1. 规划虚拟中心的轨迹 $\mathbf{p}_v(t)$
2. 每架 UAV 跟踪其在虚拟结构中的期望位置
3. 虚拟中心通过反馈调整，保持编队形状

**优缺点：**

| 优点 | 缺点 |
|------|------|
| 编队形状保持精确 | 虚拟结构是刚性约束 |
| 整体协调性好 | 灵活性差 |
| 适合密集编队 | 编队重构需要重新规划 |

### 2.3 基于一致性 (Consensus-Based)

分布式控制，每架 UAV 仅与邻居通信，通过一致性协议达成编队协调。

```
  ○ ─── ○ ─── ○ ─── ○
  │           │       │
  └───── ○ ──┘       │
              │       │
              └── ○ ──┘
  (通信拓扑图)
```

**一致性协议：**

$$
\dot{\mathbf{p}}_i = \mathbf{v}_i
$$

$$
\dot{\mathbf{v}}_i = -\sum_{j \in \mathcal{N}_i} \left[ k_p (\mathbf{p}_i - \mathbf{p}_j - \mathbf{d}_{ij}^{des}) + k_d (\mathbf{v}_i - \mathbf{v}_j) \right]
$$

其中 $\mathcal{N}_i$ 是第 $i$ 架 UAV 的邻居集合，$\mathbf{d}_{ij}^{des}$ 是期望的相对位置。

## 3. 通信拓扑及其影响

### 3.1 通信拓扑的图论表示

通信拓扑用有向图 $\mathcal{G} = (\mathcal{V}, \mathcal{E})$ 表示：
- $\mathcal{V} = \{1, 2, \ldots, N\}$：UAV 节点集合
- $\mathcal{E} \subseteq \mathcal{V} \times \mathcal{V}$：通信边集合

**邻接矩阵 $\mathbf{A}$：**

$$
A_{ij} = \begin{cases} 1 & \text{如果 } j \in \mathcal{N}_i \\ 0 & \text{否则} \end{cases}
$$

**度矩阵 $\mathbf{D}$：** 对角矩阵，$D_{ii} = \sum_j A_{ij}$

**拉普拉斯矩阵 $\mathbf{L} = \mathbf{D} - \mathbf{A}$**

### 3.2 常见通信拓扑

```
全连接图              环形图              星形图              链式图
  ○─○               ○─○               ○                 ○─○─○─○
  │╲│               │   │             ╱│╲
  ○─○               ○─○           ○─○─○─○
```

| 拓扑类型 | 通信量 | 鲁棒性 | 收敛速度 | 适用规模 |
|---------|--------|--------|---------|---------|
| 全连接 | O(N²) | 最强 | 最快 | < 5 |
| 环形 | O(N) | 一般 | 中等 | 10–50 |
| 星形 | O(N) | 弱(中心故障) | 快 | < 10 |
| 链式 | O(N) | 弱(单点断裂) | 慢 | 不推荐 |
| 随机图 | O(N log N) | 较强 | 较快 | > 10 |

### 3.3 通信约束的影响

| 通信约束 | 对编队的影响 | 解决方案 |
|---------|------------|---------|
| 延迟 | 振荡、不稳定 | 延迟补偿、鲁棒控制 |
| 丢包 | 信息缺失、性能下降 | 估计器、冗余通信 |
| 带宽限制 | 更新率低 | 事件触发通信 |
| 拓扑切换 | 暂态不稳定 | 切换系统理论 |

## 4. 避碰集成

### 4.1 避碰约束

在编队飞行中，需要确保 UAV 之间以及与障碍物之间保持安全距离：

$$
\|\mathbf{p}_i - \mathbf{p}_j\| \geq d_{safe}, \quad \forall i \neq j
$$

### 4.2 避碰控制方法

**排斥势场法：**

$$
\mathbf{f}_{rep,ij} = \begin{cases} k_{rep} \left(\frac{1}{\|\mathbf{d}_{ij}\|} - \frac{1}{d_0}\right) \frac{\mathbf{d}_{ij}}{\|\mathbf{d}_{ij}\|^2} & \text{if } \|\mathbf{d}_{ij}\| < d_0 \\ \mathbf{0} & \text{otherwise} \end{cases}
$$

其中 $\mathbf{d}_{ij} = \mathbf{p}_i - \mathbf{p}_j$，$d_0$ 为影响范围。

**CBF (Control Barrier Function) 方法：**

定义安全约束 $h(\mathbf{p}_i, \mathbf{p}_j) = \|\mathbf{p}_i - \mathbf{p}_j\|^2 - d_{safe}^2 \geq 0$

约束条件：$\dot{h} + \gamma h \geq 0$

### 5.3 编队控制器 MATLAB Function

```matlab
function u_formation = formation_controller(states, desired_offsets, ...
                                            leader_state, topology, gains)
% 多 UAV 编队控制器
% states: Nx1 struct array，每个 UAV 的状态
% desired_offsets: Nx3 矩阵，期望编队偏移
% topology: NxN 邻接矩阵
% gains: 结构体 .kp, .kd

    N = length(states);
    u_formation = zeros(N, 3);

    for i = 1:N
        if i == leader_id
            % Leader 使用轨迹跟踪控制
            u_formation(i,:) = trajectory_tracker(states(i), ref_traj);
        else
            % Follower 使用一致性协议
            force = [0, 0, 0];
            for j = 1:N
                if topology(i, j) > 0
                    dp = states(i).pos - states(j).pos;
                    dv = states(i).vel - states(j).vel;
                    d_des = desired_offsets(i,:) - desired_offsets(j,:);
                    force = force - gains.kp * (dp - d_des) - gains.kd * dv;
                end
            end

            % 避碰力
            for j = 1:N
                if i ~= j
                    d = states(i).pos - states(j).pos;
                    dist = norm(d);
                    if dist < d_safe * 2
                        force = force + k_rep * d / dist^3;
                    end
                end
            end

            u_formation(i,:) = force;
        end
    end
end
```

## 6. 编队重构

### 6.1 重构场景

| 场景 | 触发条件 | 行为 |
|------|---------|------|
| 新机加入 | 通信发现 | 重新计算编队位置 |
| 成员退出 | 故障/电量低 | 缩小编队或重新分配 |
| 编队变换 | 任务需求 | 平滑过渡到新队形 |
| 避障重构 | 障碍物检测 | 暂时打破队形后恢复 |

### 6.2 平滑编队变换

使用 B 样条或最小 snap 轨迹规划实现编队变换的平滑过渡：

$$
\mathbf{p}_i(t) = (1 - \alpha(t)) \mathbf{p}_i^{old} + \alpha(t) \mathbf{p}_i^{new}
$$

其中 $\alpha(t)$ 是从 0 到 1 的平滑过渡函数（如 5 次多项式）。

## 思考题

1. **领航-跟随架构中，如果领航者突然做急转弯机动，跟随者会面临什么问题？如何改善？**

<details><summary>参考答案</summary>
跟随者面临的主要问题：(1) **延迟响应**——跟随者通过通信获取领航者状态存在延迟，急转弯时延迟导致跟随者滞后；(2) **过冲**——急转弯产生的横向加速度可能导致跟随者过冲，偏离期望编队位置；(3) **通信中断**——急转弯可能超出通信范围或导致遮挡。改善方法：(1) **预测前馈**——跟随者不仅跟踪领航者当前位置，还预测其未来运动；(2) **路径协调**——领航者在做机动前通知跟随者预调整；(3) **鲁棒控制**——使用滑模控制或自适应控制处理不确定性和延迟；(4) **降低速度**——急转弯时自动降低编队整体速度。
</details>

2. **一致性协议中，拉普拉斯矩阵的特征值如何影响编队的收敛速度？什么样的通信拓扑收敛最快？**

<details><summary>参考答案</summary>
一致性协议的收敛速度由拉普拉斯矩阵 $\mathbf{L}$ 的次小特征值 $\lambda_2$（代数连通度）决定。$\lambda_2$ 越大，收敛越快。全连接图的 $\lambda_2 = N$（最大值），因此收敛最快。环形图的 $\lambda_2 = 2(1 - \cos(2\pi/N)) \approx (2\pi/N)^2$，当 $N$ 大时 $\lambda_2$ 很小，收敛慢。最大特征值 $\lambda_N$ 与最小特征值 $\lambda_2$ 的比值 $\lambda_N/\lambda_2$（谱比率）决定了算法的数值稳定性，越小越好。因此，设计通信拓扑时应最大化 $\lambda_2$，同时最小化 $\lambda_N/\lambda_2$。
</details>

3. **在 Simulink 中仿真 100 架 UAV 的编队飞行，计算复杂度主要在哪里？有什么优化策略？**

<details><summary>参考答案</summary>
计算复杂度主要在：(1) **编队控制器**——$O(N^2)$ 的两两交互计算；(2) **避碰检测**——$O(N^2)$ 的距离检查；(3) **通信模拟**——每个时间步需要交换 $N$ 个状态。优化策略：(1) **空间分区**——使用 KD-tree 或网格分区，只检查临近 UAV，将避碰检测降到 $O(N \log N)$；(2) **分层控制**——将编队分为小组，小组内一致性 + 小组间虚拟结构；(3) **仿真实现**——使用 MATLAB 数组运算代替 for 循环，利用 SIMD 并行；(4) **可变步长**——编队控制器使用较大步长（如 0.1s），动力学使用小步长（如 0.001s）。
</details>

4. **编队飞行中如何处理部分 UAV 失联或电量不足的情况？**

<details><summary>参考答案</summary>
需要设计**容错编队策略**：(1) **心跳检测**——每架 UAV 周期性广播心跳，超时未收到则标记为失联；(2) **角色重分配**——失联 UAV 的编队位置由最近的邻居顶替或重新分配；(3) **编队收缩**——减少编队规模，剩余 UAV 重新计算编队形状；(4) **紧急返航**——电量不足的 UAV 自动脱离编队，按最短路径返航；(5) **降级模式**——编队从精确阵型降级为松散跟随，降低精度要求以提高鲁棒性。在 Simulink 中使用 Stateflow 实现故障状态机，管理正常→降级→紧急等状态转换。
</details>

5. **通信延迟对基于一致性的编队控制有什么影响？如何补偿？**

<details><summary>参考答案</summary>
通信延迟 $\tau$ 会导致一致性协议变为时滞系统 $\dot{\mathbf{v}}_i = -\sum_j k_p(\mathbf{p}_i(t) - \mathbf{p}_j(t-\tau))$。影响包括：(1) 收敛速度变慢；(2) 可能产生振荡甚至失稳；(3) 编队精度下降。临界稳定时滞 $\tau_{max}$ 与 $\lambda_2$ 和控制增益相关。补偿方法：(1) **Smith 预估器**——在控制回路中加入延迟模型的预测；(2) **降低增益**——牺牲收敛速度换取稳定性；(3) **事件触发通信**——只在状态变化显著时通信，减少通信频率但保证信息新鲜度；(4) **时滞相关稳定性分析**——使用 Lyapunov-Krasovskii 泛函推导稳定的增益范围。
</details>
