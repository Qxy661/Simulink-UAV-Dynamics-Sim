## 1. 传感器在 UAV 状态估计中的角色

无人机要实现自主飞行，必须实时知道自身的**位置、速度、姿态**等状态量。
这些状态无法直接测量，需要依赖多种传感器进行间接推算与融合。

```
┌─────────────────────────────────────────────────────┐
│                  无人机状态估计系统                     │
│                                                       │
│   ┌──────┐   ┌──────┐   ┌──────┐   ┌──────────┐     │
│   │ IMU  │   │ GPS  │   │气压计│   │ 磁力计   │     │
│   └──┬───┘   └──┬───┘   └──┬───┘   └────┬─────┘     │
│      │          │          │             │           │
│      ▼          ▼          ▼             ▼           │
│   ┌──────────────────────────────────────────┐       │
│   │          传感器融合滤波器 (EKF)            │       │
│   └──────────────────┬───────────────────────┘       │
│                      │                               │
│                      ▼                               │
│   ┌──────────────────────────────────────────┐       │
│   │  状态输出: 位置 / 速度 / 姿态 / 偏置     │       │
│   └──────────────────────────────────────────┘       │
└─────────────────────────────────────────────────────┘
```

**传感器分类：**

| 传感器 | 测量量 | 频率范围 | 特点 |
|--------|--------|----------|------|
| IMU (加速度计) | 比力 (specific force) | 100–1000 Hz | 短期精确，长期漂移 |
| IMU (陀螺仪) | 角速度 | 100–1000 Hz | 短期精确，零偏漂移 |
| GPS | 位置、速度 | 1–10 Hz | 长期稳定，低频、有延迟 |
| 气压计 | 大气压力 (高度) | 10–50 Hz | 受天气影响，漂移 |
| 磁力计 | 地磁场方向 | 10–100 Hz | 受铁磁干扰 |
| 光流相机 | 角速度/相对位移 | 30–60 Hz | 依赖纹理与光照 |

## 2. IMU 建模

### 2.1 加速度计模型

MEMS 加速度计测量的是**比力** (specific force)，即物体的非引力加速度：

$$
\mathbf{a}_{meas} = \mathbf{R}^T (\mathbf{a}_{true} - \mathbf{g}) + \mathbf{b}_a + \mathbf{n}_a
$$

其中：
- $\mathbf{a}_{true}$：真实加速度（惯性系）
- $\mathbf{g} = [0, 0, -g]^T$：重力加速度
- $\mathbf{R}$：从惯性系到机体系的旋转矩阵
- $\mathbf{b}_a$：加速度计零偏 (bias)
- $\mathbf{n}_a$：测量噪声

**噪声组成：**

| 噪声类型 | 符号 | 单位 | 物理含义 |
|----------|------|------|----------|
| 白噪声 (White Noise) | $\sigma_{wn}$ | m/s^2/√Hz | 高频随机抖动 |
| 零偏不稳定性 (Bias Instability) | $\sigma_{bi}$ | m/s^2 | 零偏的缓慢变化 |
| 随机游走 (Random Walk) | $\sigma_{rw}$ | m/s^3/√Hz | 零偏变化率的随机性 |

**Simulink 实现：**

```
┌─────────────────────────────────────────────┐
│            加速度计 Simulink 模型             │
│                                             │
│  a_true ──►[R^T*(a-g)]──►(+)──► a_meas     │
│                               ▲             │
│              b_a ─────────────┤             │
│              n_a ─────────────┘             │
│                                             │
│  零偏动力学: b_a_dot = n_rw (随机游走)       │
└─────────────────────────────────────────────┘
```

### 2.2 陀螺仪模型

MEMS 陀螺仪测量机体相对于惯性空间的角速度在机体系下的投影：

$$
\boldsymbol{\omega}_{meas} = \boldsymbol{\omega}_{true} + \mathbf{b}_g + \mathbf{n}_g
$$

其中：
- $\boldsymbol{\omega}_{true}$：真实角速度
- $\mathbf{b}_g$：陀螺仪零偏
- $\mathbf{n}_g$：陀螺仪白噪声

**零偏动力学 (一阶高斯-马尔可夫过程)：**

$$
\dot{\mathbf{b}}_g = -\frac{1}{\tau_g} \mathbf{b}_g + \mathbf{w}_g
$$

其中 $\tau_g$ 为相关时间，$\mathbf{w}_g$ 为驱动噪声。

### 2.3 Allan 方差与噪声参数辨识

Allan 方差是辨识 IMU 噪声参数的标准方法，用于从实测数据中提取各项噪声指标。

**原理：**
对 IMU 输出进行不同采样时长 $\tau$ 的分组平均，计算组间差值的方差：

$$
\sigma^2(\tau) = \frac{1}{2(N-1)} \sum_{k=1}^{N-1} (\bar{y}_{k+1} - \bar{y}_k)^2
$$

**Allan 方差双对数图解读：**

```
log(σ²)
  │
  │  斜率 -1: 角度随机游走 (ARW)
  │  ╲
  │    ╲
  │      ╲
  │        ╲___________   斜率 0: 零偏不稳定性 (BI)
  │                      ╲
  │                        ╲  斜率 +1: 速率随机游走 (RRW)
  │
  └──────────────────────────── log(τ)
```

**辨识步骤：**

1. 在恒温环境下静态采集 IMU 数据（数小时）
2. 计算 Allan 方差曲线
3. 拟合各段斜率，提取噪声参数：

| 噪声参数 | Allan 方差斜率 | 提取方式 |
|----------|---------------|----------|
| 角度随机游走 ARW | -1 (斜率) | τ=1s 处读值 |
| 零偏不稳定性 BI | 0 (谷底) | 最小值 |
| 速率随机游走 RRW | +1 (斜率) | 斜率拟合 |

**MATLAB Allan 方差计算示例：**

```matlab
% 读取静态IMU数据
gyro_data = load('static_gyro.mat');  % rad/s
fs = 200;  % 采样频率 Hz

% 计算Allan方差
[avar, tau] = allanvar(gyro_data, 'octave', fs);

% 双对数图
loglog(tau, sqrt(avar));
xlabel('\tau (s)'); ylabel('\sigma(\tau) (rad/s)');
title('陀螺仪 Allan 方差');
grid on;

% 提取ARW (τ=1s处)
ARW = interp1(tau, sqrt(avar), 1);  % rad/s/√Hz
```

## 3. GPS 建模

### 3.1 GPS 测量模型

GPS 接收机直接输出**位置**和**速度**，但精度受多种误差源影响。

**位置测量模型：**

$$
\mathbf{p}_{GPS} = \mathbf{p}_{true} + \mathbf{b}_{GPS} + \mathbf{n}_{GPS}
$$

**速度测量模型：**

$$
\mathbf{v}_{GPS} = \mathbf{v}_{true} + \mathbf{b}_{v,GPS} + \mathbf{n}_{v,GPS}
$$

### 3.2 GPS 误差特性

| 误差源 | 典型量级 | 特性 |
|--------|---------|------|
| 卫星钟差 | 1–3 m | 差分可消除 |
| 电离层延迟 | 2–10 m | 双频可消除 |
| 对流层延迟 | 0.5–2 m | 模型可部分消除 |
| 多路径效应 | 0.5–5 m | 环境相关 |
| 几何精度因子 (DOP) | 乘以基础误差 | 与卫星几何相关 |

**精度等级：**

| GPS 类型 | 水平精度 | 垂直精度 | 更新率 |
|----------|---------|---------|--------|
| 标准单点 (SPS) | 2–5 m | 5–10 m | 1–5 Hz |
| DGPS | 0.5–2 m | 1–3 m | 1–5 Hz |
| RTK | 1–2 cm | 2–5 cm | 10–20 Hz |
| PPP | 5–10 cm | 10–20 cm | 1 Hz |

### 3.3 GPS 延迟建模

GPS 存在**输出延迟**，在融合滤波器中必须考虑：

```
时间轴 ──────────────────────────────────►
          │         │
    IMU   │  真实   │  GPS
    测量  │  状态   │  输出
    t_k   │  t_k    │  t_k - Δt_gps
          │         │
          └── Δt_gps ─┘
```

延迟补偿方法：
1. **状态缓冲法**：保存历史状态，GPS 到达时回溯到对应时刻
2. **测量预测法**：将 GPS 测量向前预测 Δt 秒

```

│                GPS 传感器 Simulink 模型                │
│                                                      │
│  p_true ──►[Delay Δt]──►(+)──► p_GPS                │
│                          ▲                           │
│          ┌───────────────┤                           │
│          │  b_GPS ───────┘ (慢变偏置)                 │
│          │                                            │
│          │  n_GPS ────────► 白噪声 (σ_pos)            │
│          │                                            │
│  v_true ──►(+)──► v_GPS                               │
│          ▲                                            │
│  n_v_GPS ┘   (σ_vel < σ_pos)                         │
└──────────────────────────────────────────────────────┘
```

## 4. 气压计建模

### 4.1 测量原理

气压计通过测量大气压力推算高度，基于国际标准大气 (ISA) 模型：

$$
h = \frac{T_0}{L} \left[ 1 - \left( \frac{P}{P_0} \right)^{\frac{LR}{gM}} \right]
$$

简化线性近似（对流层内）：

$$
h \approx \frac{P_0 - P}{\rho_0 g} + h_{bias}
$$

### 4.2 误差特性

| 误差源 | 量级 | 说明 |
|--------|------|------|
| 传感器白噪声 | 0.1–0.5 m | 高频随机 |
| 温度漂移 | 1–5 m | 与环境温度相关 |
| 天气系统偏差 | 10–50 m | 气压变化导致 |
| 建筑物效应 | 2–10 m | 风吹过建筑物造成局部气压变化 |

### 4.3 气压计 Simulink 模型

```matlab
% 气压计参数
P0 = 101325;      % 海平面标准气压 Pa
rho0 = 1.225;     % 海平面空气密度 kg/m^3
g = 9.81;         % 重力加速度 m/s^2
noise_std = 0.3;  % 噪声标准差 m
drift_rate = 0.01; % 漂移率 m/s

% 高度计算
h_baro = (P0 - P_measured) / (rho0 * g);
h_baro_noisy = h_baro + noise_std * randn + drift;
```

## 5. 磁力计建模

### 5.1 测量原理

磁力计测量地磁场在机体系下的分量，用于推算航向角 (heading)。

$$
\mathbf{m}_{meas} = \mathbf{R}_{soft} \mathbf{R}_{hard} (\mathbf{R}^T \mathbf{m}_{earth} + \mathbf{m}_{local}) + \mathbf{n}_m
$$

其中：
- $\mathbf{m}_{earth}$：地磁场向量（北向分量 + 垂直分量）
- $\mathbf{R}_{hard}$：硬铁校正矩阵
- $\mathbf{R}_{soft}$：软铁校正矩阵
- $\mathbf{m}_{local}$：机体硬铁偏移
- $\mathbf{n}_m$：测量噪声

### 5.2 硬铁与软铁效应

**硬铁效应 (Hard Iron)：**
由永磁体或直流电流产生的恒定磁场偏移。

$$
\mathbf{m}_{hard} = \begin{bmatrix} b_x \\ b_y \\ b_z \end{bmatrix} \quad \text{(常数偏移向量)}
$$

**软铁效应 (Soft Iron)：**
由铁磁材料对地磁场的扭曲/放大效应。

$$
\mathbf{R}_{soft} = \begin{bmatrix} s_{xx} & s_{xy} & s_{xz} \\ s_{yx} & s_{yy} & s_{yz} \\ s_{zx} & s_{zy} & s_{zz} \end{bmatrix} \quad \text{(3×3 校正矩阵)}
$$

### 5.3 磁力计校准

校准步骤：
1. 将无人机在三维空间中缓慢旋转，采集各方向磁力计数据
2. 拟合椭球面，提取硬铁偏移和软铁矩阵
3. 应用校正：$\mathbf{m}_{calibrated} = \mathbf{R}_{soft}^{-1} (\mathbf{m}_{raw} - \mathbf{m}_{hard})$

```matlab
% 磁力计椭球拟合校准
raw_data = load('mag_calibration.mat');  % Nx3 矩阵
% 椭球拟合
center = mean(raw_data);
A = (raw_data - center)' * (raw_data - center);
[V, D] = eig(A);
scale = sqrt(diag(D));
R_soft_inv = V * diag(1./scale) * V';
m_cal = (raw_data - center) * R_soft_inv';
```

### 5.4 航向角计算

经校准的磁力计数据计算航向：

$$
\psi_{mag} = \text{atan2}(-m_y \cos\phi + m_z \sin\phi, \; m_x \cos\theta + m_y \sin\theta \sin\phi + m_z \sin\theta \cos\phi)
$$

## 7. Simulink 传感器噪声建模实战

### 7.1 使用 Band-Limited White Noise 模块

Simulink 的 **Band-Limited White Noise** 模块是模拟传感器白噪声的标准工具。

**关键参数：**
- **Noise power**：噪声功率谱密度 $S_0$ (单位: 单位²/Hz)
- **Sample time**：采样时间（应远小于系统最小时间常数）

**转换公式：**
给定连续时间噪声标准差 $\sigma$ 和采样时间 $T_s$：

$$
S_0 = \frac{\sigma^2}{T_s}
$$

### 7.2 传感器模型封装示例

```matlab

IMU.accel.bias_instability = 0.001; % m/s^2 (零偏不稳定性)
IMU.accel.random_walk = 0.0001;   % m/s^3/√Hz
```

## 8. 传感器融合的动机

### 8.1 单一传感器的局限性

```
精度
 ▲
 │      GPS ──────────── 长期稳定
 │        ╲
 │          ╲
 │            ╲
 │              ╲   融合输出
 │                ╲──────────
 │  IMU 积分 ──── ╱
 │           ╱
 │         ╱
 │       ╱
 │     ╱   短期精确
 │
 └──────────────────────────────► 时间
```

- **IMU**：短期精度高，但积分漂移无界
- **GPS**：长期稳定，但更新慢、噪声大
- **气压计**：垂直精度好，但受天气影响
- **磁力计**：提供航向参考，但受磁干扰

### 8.2 互补特性

| 传感器 | 高频特性 | 低频特性 | 更新率 |
|--------|---------|---------|--------|
| IMU | 优秀 | 差 (漂移) | 高 (200–1000 Hz) |
| GPS | 差 (噪声) | 优秀 | 低 (1–10 Hz) |
| 气压计 | 中等 | 中等 | 中 (25 Hz) |
| 磁力计 | 中等 | 中等 | 中 (50 Hz) |

传感器融合的核心思想：**取长补短**，利用各传感器的互补特性，获得优于任何单一传感器的状态估计。

## 思考题

1. **为什么 MEMS 加速度计测量的是比力而非真正的加速度？在静止悬停时，加速度计的输出是多少？**

<details><summary>参考答案</summary>
加速度计的工作原理基于检测质量块 (proof mass) 相对于外壳的位移。当加速度计处于自由落体状态时，质量块与外壳无相对位移，输出为零。因此加速度计测量的是除重力以外的非引力加速度（比力）。在静止悬停时，无人机需要向上的推力抵消重力，加速度计测量到向上的比力，输出为 $[0, 0, +g]^T$（在机体系中），即一个向上的 1g 加速度。这也是为什么在 EKF 中，加速度计可以作为姿态的观测量——通过测量比力方向来推断重力方向。
</details>

2. **Allan 方差曲线的斜率分别对应哪些噪声类型？如果 IMU 的 Allan 方差在 τ=1s 处读值为 0.003 °/√s，这意味着什么？**

<details><summary>参考答案</summary>
斜率 -1 对应角度随机游走 (Angle Random Walk)，斜率 0 对应零偏不稳定性 (Bias Instability)，斜率 +1 对应速率随机游走 (Rate Random Walk)。τ=1s 处读值为 0.003 °/√s 表示陀螺仪的角度随机游走系数为 0.003 °/√s（即 ARW = 0.003 °/√s）。这意味着陀螺仪在 1 秒的积分过程中，角度误差的标准差约为 0.003°；在 100 秒积分后，角度误差约为 0.03°。这是衡量陀螺仪短期精度的关键指标。
</details>

3. **GPS 输出延迟为 200ms，IMU 频率为 200Hz。如何在 EKF 中处理这个延迟？如果忽略延迟会有什么后果？**

<details><summary>参考答案</summary>
方法一：**状态缓冲法**——维护一个长度为 200×0.2=40 帧的环形缓冲区，保存最近 40 个 IMU 时刻的状态估计和协方差。当 GPS 测量到达时，从缓冲区中找到对应时刻的状态（40帧前），在该点执行 EKF 更新，然后将更新后的状态向前传播到当前时刻。方法二：**测量预测法**——将 GPS 测量加上延迟期间的速度变化量，补偿到当前时刻。如果忽略延迟，状态估计会产生系统性偏差，在快速机动时误差更显著，可能导致控制器跟踪性能下降甚至不稳定。
</details>

4. **磁力计的硬铁效应和软铁效应分别是什么？为什么每次更换机体安装位置都需要重新校准磁力计？**

<details><summary>参考答案</summary>
硬铁效应是由机体上的永磁体或直流导线产生的恒定磁场偏移，表现为磁力计测量的椭球中心偏离原点。软铁效应是由铁磁材料（如电机、电池外壳）对地磁场的扭曲和放大，表现为椭球的形状和方向发生变化。更换安装位置后，磁力计周围铁磁材料的分布改变，硬铁偏移量和软铁扭曲矩阵都会变化，因此需要重新校准。此外，电源线的电流变化也会引起动态硬铁效应，这也是为什么在不同电流状态下磁力计读数会有差异。
</details>

5. **设计一个传感器故障检测机制：如何判断 GPS 数据是否可信？**

<details><summary>参考答案</summary>
可以设计多层检测机制：(1) **量程检查**——位置是否超出合理范围，速度是否超过最大可能值；(2) **一致性检查**——将 GPS 位置与 IMU 积分位置比较，如果残差超过阈值（如 3σ）则标记异常；(3) **NMEA/协议检查**——检查 GPS 解的状态标志、卫星数量、DOP 值；(4) **时间连续性检查**——相邻 GPS 输出之间的加速度是否合理；(5) **卡方检验**——对 EKF 的创新序列 (innovation) 进行卡方检验，异常大的创新表示传感器可能故障。在 Simulink 中可用 Stateflow 实现故障状态机，在检测到故障时切换到纯 IMU 积分模式。
</details>
