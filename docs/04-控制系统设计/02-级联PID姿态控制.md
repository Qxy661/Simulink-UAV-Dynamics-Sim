# 级联 PID 姿态控制

> 预计阅读：25 分钟 | 前置知识：PID 控制基础、四旋翼动力学、坐标系与欧拉角

---

## 1. 级联控制架构概述

四旋翼无人机是一个欠驱动系统（6 自由度、4 个控制输入），直接对位置进行 PID 控制难以保证稳定性和响应速度。工程实践中普遍采用**级联 PID（Cascade PID）**控制架构，将控制任务分解为多个嵌套的控制回路。

**核心思想**：内环控制快速动态（角速率），外环控制慢速动态（位置），通过带宽分离保证各环独立工作。

```
带宽关系：角速率环 >> 姿态角环 >> 位置环
典型值：   50~200 rad/s    10~50 rad/s    1~10 rad/s
```

## 2. 完整级联结构

### 2.1 三层嵌套架构

```
位置环（最外层）  →  姿态环（中间层）  →  角速率环（最内层）
  ~1-10 rad/s         ~10-50 rad/s         ~50-200 rad/s
  更新 50-100 Hz      更新 100-400 Hz      更新 400-1000 Hz
```

## 3. 各环路详细设计

### 3.1 位置环（Position Loop）

位置环是最外层的控制回路，负责将期望位置转化为期望姿态角。

**输入**：期望位置 (x_ref, y_ref, z_ref) 与实际位置 (x, y, z) 的误差
**输出**：期望姿态角 (φ_des, θ_des) 和总推力 T

```
x 方向：误差 ex = x_ref - x  →  PID_x  →  期望俯仰角 θ_des
y 方向：误差 ey = y_ref - y  →  PID_y  →  期望滚转角 φ_des
z 方向：误差 ez = z_ref - z  →  PID_z  →  总推力 T
```

**坐标变换考虑**：期望姿态角需要根据偏航角进行坐标旋转：

```
φ_des =  PID_x_output · cos(ψ) + PID_y_output · sin(ψ)
θ_des = -PID_x_output · sin(ψ) + PID_y_output · cos(ψ)
```

**位置环典型参数：**

| 轴 | Kp | Ki | Kd | 备注 |
|---|---|---|---|---|
| X | 0.5~1.5 | 0.0~0.1 | 0.2~0.8 | 水平位置 |
| Y | 0.5~1.5 | 0.0~0.1 | 0.2~0.8 | 水平位置 |
| Z | 1.0~3.0 | 0.1~0.5 | 0.3~1.0 | 垂直高度 |

### 3.2 姿态环（Attitude Loop）

姿态环是中间层，负责将期望姿态角转化为期望角速率。

**输入**：期望姿态角 (φ_des, θ_des, ψ_des) 与实际姿态角 (φ, θ, ψ) 的误差
**输出**：期望角速率 (p_des, q_des, r_des)

```
滚转：误差 eφ = φ_des - φ  →  PID_φ  →  期望滚转角速率 p_des
俯仰：误差 eθ = θ_des - θ  →  PID_θ  →  期望俯仰角速率 q_des
偏航：误差 eψ = ψ_des - ψ  →  PID_ψ  →  期望偏航角速率 r_des
```

**姿态环典型参数：**

| 轴 | Kp | Ki | Kd | 备注 |
|---|---|---|---|---|
| Roll (φ) | 2.0~6.0 | 0.0~0.5 | 0.0~0.2 | 滚转角 |
| Pitch (θ) | 2.0~6.0 | 0.0~0.5 | 0.0~0.2 | 俯仰角 |
| Yaw (ψ) | 1.5~4.0 | 0.0~0.3 | 0.0~0.1 | 偏航角 |

### 3.3 角速率环（Rate Loop）

角速率环是最内层的控制回路，也是响应最快的环路。

**输入**：期望角速率 (p_des, q_des, r_des) 与实际角速率 (p, q, r) 的误差
**输出**：三个轴的力矩指令 (τx, τy, τz)

```
滚转速率：误差 ep = p_des - p  →  PID_p  →  滚转力矩 τx
俯仰速率：误差 eq = q_des - q  →  PID_q  →  俯仰力矩 τy
偏航速率：误差 er = r_des - r  →  PID_r  →  偏航力矩 τz
```

**角速率环典型参数：**

| 轴 | Kp | Ki | Kd | 备注 |
|---|---|---|---|---|
| Roll Rate (p) | 0.5~2.0 | 0.1~1.0 | 0.001~0.02 | 采样率 ≥ 400 Hz |
| Pitch Rate (q) | 0.5~2.0 | 0.1~1.0 | 0.001~0.02 | 采样率 ≥ 400 Hz |
| Yaw Rate (r) | 0.5~1.5 | 0.05~0.5 | 0.0~0.01 | 采样率 ≥ 400 Hz |

## 4. 带宽分离原则

### 4.1 为什么要分离带宽

级联控制的核心假设是：**内环的响应速度远快于外环**。这样外环可以将内环视为理想的"跟踪器"。

```
若内环带宽 = ω_inner，外环带宽 = ω_outer

则需满足：ω_inner ≥ (5 ~ 10) · ω_outer
```

### 4.2 带宽不满足的后果

| 情况 | 表现 | 后果 |
|---|---|---|
| 内环太慢 | 外环指令内环跟不上 | 系统振荡、不稳定 |
| 内外环带宽接近 | 环路耦合 | 互相干扰、性能下降 |
| 正确分离 | 内环快速跟踪，外环平稳控制 | 良好动态性能 |

### 4.3 带宽设计指南

```
                    频率 (rad/s)
    0.1     1       10      100     1000
    ├───────┼───────┼───────┼───────┤
    │  位置环  │  姿态环  │  角速率环  │  ← 各环带宽
    │ 1~10   │ 10~50  │ 50~200  │
    │        │        │         │
    │◄─10倍──►│◄──10倍──►│
```

## 5. 电机混控器（Motor Mixer）

### 5.1 混控矩阵

角速率环输出的力矩指令和位置环输出的推力指令，需要通过混控器分配到各个电机。

对于标准 X 构型四旋翼：

```
┌     ┐   ┌                            ┐   ┌     ┐
│ ω₁² │   │  1    -l    -l    -k/k    │   │  T  │
│ ω₂² │ = │  1    -l     l     k/k    │ × │  τx │
│ ω₃² │   │  1     l     l    -k/k    │   │  τy │
│ ω₄² │   │  1     l    -l     k/k    │   │  τz │
└     ┘   └                            ┘   └     ┘
```

其中：
- l = 机臂长度
- k = 电机力矩系数（推力与反扭矩比）
- T = 总推力，τx/τy/τz = 三轴力矩

### 5.2 四旋翼 X 构型电机布局

```
         前方 (North)
           ↑
    Motor 1 ● ──── ● Motor 2
           ╲   ╱
            ╲ ╱
             ╳  ← 机架中心
            ╱ ╲
           ╱   ╲
    Motor 4 ● ──── ● Motor 3

    M1: 逆时针(CCW)   M2: 顺时针(CW)
    M4: 顺时针(CW)    M3: 逆时针(CCW)
```

### 5.3 Simulink 混控器实现

混控器可以用一个增益矩阵（Gain Matrix）实现：

```matlab
% 混控矩阵 M (4x4)
M = [ 1, -l, -l, -k/k;
      1, -l,  l,  k/k;
      1,  l,  l, -k/k;
      1,  l, -l,  k/k];

% 控制输入向量
u = [T; tau_x; tau_y; tau_z];

% 电机转速平方
omega_sq = M * u;

% 限幅
omega_sq = max(omega_sq, 0);        % 不能为负
omega_sq = min(omega_sq, omega_max^2); % 最大转速限制

% 实际转速
omega = sqrt(omega_sq);
```

## 6. Simulink 实现步骤

### 6.1 模型架构

```
Simulink Model
├── Position_Controller (Subsystem)
│   ├── PID_x → θ_des
│   ├── PID_y → φ_des
│   └── PID_z → T
├── Attitude_Controller (Subsystem)
│   ├── PID_roll → p_des
│   ├── PID_pitch → q_des
│   └── PID_yaw → r_des
├── Rate_Controller (Subsystem)
│   ├── PID_p → τx
│   ├── PID_q → τy
│   └── PID_r → τz
├── Motor_Mixer (Subsystem)
│   └── M * [T; τx; τy; τz] → ω₁,ω₂,ω₃,ω₄
├── Quadrotor_Dynamics (Subsystem)
│   └── 6-DOF 动力学模型
└── Sensors (Subsystem)
    └── IMU + GPS + 气压计模型
```

### 6.2 分步搭建指南

**Step 1：搭建角速率环**
1. 新建 Simulink 模型
2. 创建三个 PID Controller 分别控制 p、q、r
3. 使用 Mux 合并三个角速率误差信号
4. 连接四旋翼动力学子系统
5. 调试角速率环参数直到响应快速且无振荡

**Step 2：搭建姿态环**
1. 在角速率环外层嵌套三个 PID Controller
2. 姿态环输出作为角速率环的参考输入
3. 调试姿态环参数，确保带宽低于角速率环

**Step 3：搭建位置环**
1. 在姿态环外层嵌套三个 PID Controller
2. 位置环输出经坐标变换后作为姿态环参考
3. 调试位置环参数

**Step 4：集成电机混控**
1. 将力矩和推力指令送入混控器
2. 混控器输出连接到动力学模型的电机输入
3. 添加电机限幅和电机动力学模型

### 6.3 调试技巧

| 阶段 | 方法 | 关键指标 |
|---|---|---|
| 角速率环 | 阶跃响应测试 | 上升时间 < 20ms，无超调 |
| 姿态环 | 阶跃响应测试 | 上升时间 < 100ms，超调 < 10% |
| 位置环 | 阶跃响应测试 | 上升时间 < 1s，稳态误差 < 5cm |
| 全系统 | 轨迹跟踪测试 | 跟踪误差 < 20cm |

## 7. 整定顺序详解

### 7.1 为什么要从内环开始

```
错误顺序：位置环 → 姿态环 → 角速率环
          (外环参数依赖内环，先调外环会导致不断修改)

正确顺序：角速率环 → 姿态环 → 位置环
          (内环稳定后作为外环的"被控对象")
```

### 7.2 角速率环整定

1. 将姿态环和位置环旁路，直接给角速率环阶跃指令
2. 先调 Kp：从小值开始增大，直到出现小幅振荡，取 60%~70%
3. 再调 Ki：消除稳态误差，从小值开始，避免积分饱和
4. 最后调 Kd：减小超调，通常值很小（0.001~0.02）
5. 验证：对三个轴分别测试阶跃响应

### 7.3 姿态环整定

1. 角速率环参数固定，给姿态环阶跃指令
2. 姿态环 PID 输出即为角速率环的参考
3. Kp 决定姿态跟踪的快速性
4. Ki 消除姿态稳态误差
5. Kd 提供阻尼，减小超调

### 7.4 位置环整定

1. 姿态环参数固定，给位置环阶跃指令
2. 位置环 PID 输出经坐标变换后作为姿态参考
3. 注意限制期望姿态角的范围（通常 ±30°）
4. 高度环（Z 轴）通常独立整定

## 8. 高级话题

### 8.1 前馈补偿

在位置环中加入速度前馈，可以改善轨迹跟踪性能：

```
θ_des = PID_x(ex) + Kff · ẋ_ref
```

其中 Kff 为前馈增益，ẋ_ref 为期望速度。

### 8.2 串级 PID 的等效传递函数分析

将内环闭环传递函数近似为一阶惯性环节：

```
G_inner(s) ≈ ωn / (s + ωn)
```

其中 ωn 为内环自然频率。外环设计时将内环视为带宽为 ωn 的"黑盒"。

### 8.3 增益调度（Gain Scheduling）

四旋翼在不同飞行状态下（悬停 vs 高速飞行），最优 PID 参数可能不同：

| 飞行状态 | Kp (姿态) | Ki (姿态) | Kd (姿态) |
|---|---|---|---|
| 悬停 | 4.0 | 0.2 | 0.1 |
| 低速巡航 | 3.5 | 0.3 | 0.15 |
| 高速飞行 | 5.0 | 0.4 | 0.08 |

## 思考题

**1.** 为什么四旋翼的位置环输出需要根据偏航角 ψ 进行坐标变换？如果不做这个变换，在偏航 90° 时会出现什么问题？

<details>
<summary>参考答案</summary>

四旋翼的"前后左右"是相对于机体坐标系的。当偏航角为 ψ 时，世界坐标系的 X 方向在机体系中对应：
- 机体系 X 分量 = cos(ψ)·X_world + sin(ψ)·Y_world
- 机体系 Y 分量 = -sin(ψ)·X_world + cos(ψ)·Y_world

如果不做变换，当 ψ = 90° 时，世界坐标系 X 方向的误差会错误地映射到机体 Y 轴（俯仰），导致无人机向侧面飞行而不是向前飞行，最终可能失控。
</details>

**2.** 角速率环的采样频率为什么要比姿态环和位置环高得多？如果三个环路使用相同的采样频率会有什么后果？

<details>
<summary>参考答案</summary>

角速率环控制的是快速动态，四旋翼的角速率变化频率可达数十 Hz，根据 Nyquist 定理，采样频率至少需要是信号最高频率的 2 倍以上。实际上通常需要 10~20 倍的过采样。

如果使用相同的低采样频率（如 50 Hz）：
- 角速率环无法快速响应，导致姿态控制延迟
- 延迟引起的相位滞后会减小相位裕度，可能导致振荡
- 无法有效抑制外部扰动（如阵风）
- 系统等效带宽降低，飞行性能下降
</details>

**3.** 混控器中出现 ω² < 0 的情况说明什么？工程上如何处理？

<details>
<summary>参考答案</summary>

ω² < 0 说明当前力矩/推力需求超出了四旋翼的物理能力。例如：
- 需要的滚转力矩太大，而对角电机已满转速
- 总推力需求与力矩需求冲突

工程处理：
1. 将负值截断为 0（该电机停转）
2. 同时对其他电机做等比例缩放，保持力矩分配比例
3. 使用优化方法（如加权最小二乘）重新分配控制量
4. 限制期望姿态角范围，避免产生过大的力矩需求
</details>

**4.** 前馈补偿 Kff 如何改善轨迹跟踪性能？为什么仅靠 PID 反馈无法达到同样效果？

<details>
<summary>参考答案</summary>

PID 反馈是被动的——只有当误差产生后才开始纠正在。对于高速轨迹跟踪，当参考信号快速变化时，反馈控制始终"滞后"。

前馈补偿利用参考信号的导数（速度）提前施加控制：
- PID：u = Kp·e + Ki·∫e + Kd·ė  （依赖误差存在）
- PID + FF：u = Kp·e + Ki·∫e + Kd·ė + Kff·ẋ_ref  （提前补偿）

前馈在参考信号变化前就开始"推动"系统，减小跟踪延迟。理想情况下，如果模型完美，前馈可以完全消除跟踪误差，反馈仅用于抑制扰动。
</details>

**5.** 在风扰动下，级联 PID 控制器中哪个环路最容易受到影响？如何增强抗风能力？

<details>
<summary>参考答案</summary>

位置环最容易受风扰动影响，因为：
- 风直接作用在机体上产生力和力矩
- 位置环是最外层、响应最慢的环路
- 风引起的位移需要较长时间才能恢复

增强抗风能力的方法：
1. 增大位置环 Kd（提供速度阻尼）
2. 在位置环加入风扰动估计（如使用 EKF 估计风速并前馈补偿）
3. 使用角速率环的高带宽快速抑制风力矩扰动
4. 考虑使用更先进的控制方法（如 MPC 或滑模控制）
</details>
